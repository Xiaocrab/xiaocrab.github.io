茫茫網誌系統中遇見Hexo

發表於 2017-01-28 | 分類於 IT | 閱讀次數
快速、簡單且強大的網誌框架
——Hexo.io

緣起

前一段時間，我的 VPS 沒有及時交費，結果被強制停機。續費之後，訪問我的 WordPress 網站結果提示db connection error錯誤。。。Oh，該死的數據庫！加之 WordPress 是出了名的「臃腫」，訪問的速度非常慢。我便心生了換一個網誌系統的念頭。

尋找

既然是重新尋找一個合適的網誌系統，那麽我對它的基本要求是：

  1. 開源：開源是最起碼的要求，不然還有什麼自建的意義嘞
  2. 支持Markdown：Markdown 的好處不必說，用過一次就忘不了
  3. 使用和維護簡單：寫篇文章要各種登錄，各種設置，然後三天兩頭出問題，這種系統我是受不了的

按照這個要求展開尋找之後我做了一個簡單的表格出來，全是開源的系統。

系統 環境 原生 Markdown 維護難度
WordPress PHP+DB ❤︎❤︎❤︎❤︎
Ghost Node.js+DB Yes ❤︎❤︎❤︎
Typecho PHP+DB Yes ❤︎❤︎❤︎
Jekyll Ruby Yes ❤︎❤︎❤︎
Hexo Node.js Yes ❤︎

通過以上的比較（雖然不是很準確），可以發現他們幾乎都要使用數據庫來存儲檔案，這就給維護帶來了一定的困難。

對於網誌系統來說，不用數據庫也行的，反而速度更快。網誌的內容不頻繁更新，最多一天就更新幾次，其餘的時間一直都沒變。多數使用了數據庫的網誌系統，為了加快頁面解析的速度，減小伺服器的負擔，做了「靜態化」的處理。

沒有數據庫，減少了對環境的依賴。要使用數據庫首先得安裝配置一個數據庫，然後還要連接數據庫，接著對數據庫進行操作、幾乎一切的配置和數據都存儲在數據庫裏面啊，整個系統都要依賴與數據庫才能運行。數據庫一但數據歇菜了，整個網站立馬就癱瘓！

找到了！

Hexo是一個出自台灣tommy351大神之手，基於Node.js的輕量級靜態網誌系統。

它

  1. 採用Markdown撰寫文章：撰寫方便，符合潮流
  2. 沒有使用數據庫：配置文件主要是*.yml格式，用文本編輯器就能編輯配置
  3. 靜態的系統：訪問的速度快！快！快！
  4. 資源豐富，易於維護：它的官方網站上已經提供了非常詳細的說明，還有豐富的主題和插件的支持。日常使用只需要幾條命令（甚至是全自動）就能完成文章的發佈

這簡直是一個完美的網誌系統啊！

動手！

官方網站是 https://hexo.io/ ，提供了多語種的支持。強烈推薦按照官方網站上的步驟來安裝配置Hexo，民間網誌上可能版本老舊或是作者的粗心大意會有些坑，切記，切記。安裝 Hexo，大致需要 Node.js、git 環境，沒錯就是這麽簡約！

裝好系統之後，開始安裝主題。推薦 NexT 主題，官方網站 是http://theme-next.iissnan.com/ 。製作精良，支持完善，同樣的還是建議到它的官網網站上去按照步驟做。

這樣不出意外的話，整個系統就可以開始用了。你可以把 Hexo 發佈到自己的 Github Pags、或者是自己的 VPS。

正如它的官網介紹那樣說的「快速、簡單且強大的網誌系統」就配置好了！
